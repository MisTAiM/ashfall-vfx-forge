Overview

Ashfall VFX Forge is a full-scale HTML5 Canvas animation platform built for Ashfall Idle. It provides real-time VFX preview, configuration, and code export for integrating animated item effects into the game.

The platform includes 44 individual effects across 7 categories, a built-in item database matching Ashfall's OSRS-style equipment system, and a complete code generation pipeline.

Getting Started

Upload an Item

Click the Upload button in the header to load any PNG/JPG item sprite. The image auto-scales to fit the canvas viewport while maintaining aspect ratio. Transparent PNGs work best — the VFX effects layer around the item's alpha channel.

Enable Effects

Use the left sidebar to toggle effects on/off. Click any effect row to enable it — the dot indicator shows active state. Multiple effects stack and composite together in real-time.

Configure

The right panel exposes all configurable parameters. Sliders and color pickers update the canvas in real-time. Context-sensitive controls appear based on which effects are active.

Export

Click Code to generate a JavaScript configuration object. Click Frame to download the current canvas as a PNG.

Code Export & Integration

The Code panel generates a JavaScript config object containing all active effects and their parameters. Example output:

// ASHFALL IDLE — VFX: Volcanic Cape // Generated by VFX Forge v3.0 const VOLCANIC_CAPE_VFX = { item: 'a2', effects: ["fire_embers","flame_aura","float_bob","bloom"], config: { intensity: 1.2, speed: 0.8, scale: 1, glowCol: "#ff6b35", partCol: "#ff4500", auraCol: "#ff6b35", ... } }; // Usage: applyVFX(canvas, img, VOLCANIC_CAPE_VFX);
The exported config is designed to be consumed by your game's item render pipeline. Implement an applyVFX() function that reads the config and applies the corresponding canvas operations during your game's render loop.

Performance Notes

The render loop uses requestAnimationFrame with ref-based state access to avoid React re-render overhead. Key optimizations:

Particle pool capped at 400. DPR-aware canvas sizing with setTransform reset every frame. No pixel-level manipulation (distortion effect removed for performance). Screen shake uses exponential decay (0.88 multiplier) instead of linear.

Enabling all 44 effects simultaneously is supported but not recommended for production. Use the Kill Particles button if the canvas becomes saturated.
